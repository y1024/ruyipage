Firefox 内核补丁说明
=====================

如果要访问部分带风控、反自动化或 Cloudflare 检测的网站，
仅靠普通启动参数通常还不够。

必须项
------

需要把 Firefox 内核里的 webdriver 标志改为 false。

也就是说，页面环境里与自动化暴露相关的 webdriver 标记，
不能继续保持默认自动化状态，否则很多站点仍然会识别出浏览器正在被自动化控制。

核心要求
--------

1. Firefox 内核中的 webdriver 标志必须为 false。
2. 不能只做页面级 JavaScript 覆盖。
3. 不能只依赖普通前端注入或运行时脚本绕过。
4. 需要从 Firefox 内核层或浏览器底层实现上处理这个标志。

为什么必须这样做
----------------

因为很多站点不会只检查页面脚本层。
它们会结合浏览器行为、环境特征和 webdriver 标志一起判断。

如果 webdriver 标志仍然暴露为自动化状态：

- 页面可能直接出现验证
- 某些输入框、按钮、接口不会正常放行
- Cloudflare / 风控页更容易触发
- 即使脚本本身没有报错，也可能拿不到正常页面结果

结论
----

如果你的目标站点对自动化敏感，
需要先确保 Firefox 内核里的 webdriver 标志已经被改成 false，
然后再使用 ruyipage 运行自动化脚本。

否则：

脚本即使写对了，页面也未必会按正常用户环境响应。
